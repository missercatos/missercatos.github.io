系统 Linux 文件系统结构
Linux 遵循 FHS（Filesystem Hierarchy Standard）文件系统层次结构标准。理解每个目录的用途是深入使用 Linux 的第一步。
根目录 / 下的关键目录：
  • /bin — 基础用户命令二进制文件（如 ls, cp, mv）
  • /boot — 启动加载器文件（内核 vmlinuz、initramfs、grub 配置）
  • /dev — 设备文件（sda, tty, null, zero 等）
  • /etc — 系统级配置文件（所有软件的配置几乎都在此）
  • /home — 用户主目录（每个用户的个人文件和配置）
  • /lib — 系统库文件和内核模块
  • /mnt & /media — 临时挂载点
  • /opt — 第三方附加软件包
  • /proc — 虚拟文件系统（进程和内核信息，如 /proc/cpuinfo）
  • /root — root 用户的主目录
  • /run — 运行时变量数据（tmpfs，重启后清空）
  • /sbin — 系统管理命令（需要 root 权限）
  • /sys — 内核和设备信息虚拟文件系统
  • /tmp — 临时文件（重启后可能清空）
  • /usr — 用户程序和数据（Unix System Resources）
     /usr/bin — 用户命令
     /usr/lib — 库文件
     /usr/share — 架构无关的共享数据
     /usr/local — 本地编译安装的软件
  • /var — 可变数据（日志、缓存、数据库等）
使用 man hier 可以查看 Linux 文件系统层次结构的完整手册。
系统 基础指令与指令查询
日常高频命令：
  • ls -la — 列出目录详细信息（含隐藏文件）
  • cd /path — 切换目录；cd - 返回上一个目录
  • pwd — 显示当前工作目录
  • cp -r 源 目标 — 复制文件或目录
  • mv 源 目标 — 移动/重命名
  • rm -rf 路径 — 强制递归删除（谨慎使用）
  • mkdir -p 路径 — 创建多级目录
  • cat / less / head / tail — 查看文件内容
  • grep -r "关键词" 目录 — 递归搜索文本
  • find /path -name "*.txt" — 按名称查找文件
  • chmod 755 file — 修改文件权限
  • chown user:group file — 修改文件所有者
  • ps aux | grep 进程名 — 查看进程
  • kill -9 PID — 强制终止进程
  • systemctl status/start/stop/enable 服务 — 管理系统服务
命令查询方法：
  • man 命令 — 查看完整手册（按 q 退出）
  • 命令 --help 或 命令 -h — 查看简要帮助
  • tldr 命令 — 命令速查（需安装 tldr：sudo pacman -S tldr）
  • info 命令 — GNU Info 格式文档
  • whatis 命令 — 一行描述命令用途
  • which 命令 — 查看命令的完整路径
系统 tar 压缩与解压
tar 是 Linux 最常用的归档工具，通常搭配 gzip / bzip2 / xz 等压缩算法使用。
常用操作速查：
  • 创建 tar 包：tar -cvf archive.tar /path/to/dir
  • 解压 tar 包：tar -xvf archive.tar
  • 创建 tar.gz：tar -czvf archive.tar.gz /path
  • 解压 tar.gz：tar -xzvf archive.tar.gz
  • 创建 tar.bz2：tar -cjvf archive.tar.bz2 /path
  • 解压 tar.bz2：tar -xjvf archive.tar.bz2
  • 创建 tar.xz：tar -cJvf archive.tar.xz /path
  • 解压 tar.xz：tar -xJvf archive.tar.xz
  • 查看压缩包内容：tar -tvf archive.tar.gz
  • 解压到指定目录：tar -xzvf archive.tar.gz -C /target/dir
参数含义：
  • -c create 创建 | -x extract 解压 | -t list 查看
  • -v verbose 显示过程 | -f file 指定文件名
  • -z gzip | -j bzip2 | -J xz
  • -C 指定目标目录
压缩率比较：xz > bzip2 > gzip；压缩速度：gzip > bzip2 > xz。日常使用 tar.gz 最为通用。
系统 显示管理器 — SDDM 配置
SDDM（Simple Desktop Display Manager）是 KDE Plasma 的默认显示管理器，也是最推荐的显示管理器。它负责图形登录界面和用户会话管理。
安装 SDDM：
sudo pacman -S sddm sddm-kcm（sddm-kcm 提供 KDE 设置界面的 SDDM 配置面板）
sudo systemctl enable sddm（设置开机启动）
SDDM 配置文件位置：
主配置：/etc/sddm.conf 或 /etc/sddm.conf.d/
主题目录：/usr/share/sddm/themes/
更换 SDDM 主题：
  • 从 AUR 安装主题：yay -S sddm-sugar-candy-git（推荐的美观主题）
  • 编辑 /etc/sddm.conf.d/theme.conf，设置 [Theme] Current=主题名
  • 在 KDE 系统设置 → 启动和关机 → 登录屏幕(SDDM) 中直接预览和应用主题
系统 脚本编写 — /usr/bin/ 与 ~/.local/bin/
Linux 系统中，可执行脚本可以放置在多个位置。放置位置决定了脚本的作用范围和调用方式。
脚本放置位置对比：
  • /usr/bin/ — 系统级可执行文件，所有用户可用，需要 root 权限写入
  • /usr/local/bin/ — 本地编译/手动安装的程序，优先级高于 /usr/bin
  • ~/.local/bin/ — 用户级可执行文件，仅当前用户可用（推荐自用脚本放这里）
  • ~/bin/ — 传统用户脚本目录（需确保在 PATH 中）
在 ~/.local/bin/ 中创建脚本的步骤： 
# 1. 确保目录存在并在 PATH 中
mkdir -p ~/.local/bin

# 2. 在 ~/.bashrc 或 ~/.zshrc 中添加（通常已默认包含）：
export PATH="$HOME/.local/bin:$PATH"

# 3. 创建脚本文件
nano ~/.local/bin/myscript

# 4. 脚本内容示例：
#!/bin/bash
echo "Hello, $USER! Today is $(date)"

# 5. 赋予执行权限
chmod +x ~/.local/bin/myscript

# 6. 现在可以在任何位置直接执行
myscript
在 /usr/bin/ 中创建系统级脚本： 
sudo nano /usr/bin/mysystem-script
sudo chmod +x /usr/bin/mysystem-script
系统 Bash 脚本语法结构
Bash 脚本以 #!/bin/bash（shebang）开头，以下是核心语法要素。
变量定义与使用： 
#!/bin/bash
name="Arch Linux"
echo "Welcome to $name"
echo "Path: ${HOME}"          # 花括号明确变量边界
readonly PI=3.14159           # 只读变量
条件判断： 
if [ "$1" == "hello" ]; then
    echo "参数是 hello"
elif [ -f "/etc/passwd" ]; then
    echo "/etc/passwd 存在"
else
    echo "其他情况"
fi

# 常用测试条件：
[ -f file ]  # 是否为普通文件
[ -d dir  ]  # 是否为目录
[ -z str  ]  # 字符串是否为空
[ -n str  ]  # 字符串是否非空
[ a -eq b ]  # 数值相等
循环： 
# for 循环
for i in {1..5}; do echo "Number: $i"; done

# while 循环
count=0
while [ $count -lt 5 ]; do
    echo "Count: $count"
    count=$((count + 1))
done
函数： 
greet() {
    local name="$1"            # local 限制作用域
    echo "你好, $name!"
}
greet "世界"
使用 shellcheck 工具检查脚本语法错误：sudo pacman -S shellcheck
系统 Vim — 使用方法与配置详解
Vim 是 Linux 下最强大的文本编辑器之一，几乎预装在每台 Linux 机器上。掌握 Vim 可以在任何服务器环境中高效编辑文件。
基础使用方法：
  • vim 文件名 — 打开文件（文件不存在则新建）
  • vim +行号 文件名 — 打开并跳转到指定行（如 vim +42 file.txt）
  • vim -R 文件名 或 view 文件名 — 只读模式打开
Vim 的四种模式（核心概念）：
  • 普通模式（Normal）：默认模式，用于导航和执行命令。按 Esc 从任何模式返回。
    移动：h j k l（左 下 上 右）、w 下个词、b 上个词、gg 文件首、G 文件尾
    删除：x 删字符、dd 删整行、dw 删词
    复制粘贴：yy 复制行、p 粘贴、u 撤销、Ctrl+r 重做
  • 插入模式（Insert）：正常输入文本。i 进入（光标前）、a（光标后）、o（下一行）、O（上一行）
  • 可视模式（Visual）：选择文本块。v 字符选、V 行选、Ctrl+v 块选
  • 命令模式（Command）：: 进入，执行保存/退出/搜索等。:w 保存、:q 退出、:wq 保存退出、:%s/旧/新/g 全局替换
配置文件：~/.vimrc（用户级）或 /etc/vimrc（系统级）
Vim 使用 Vimscript 语法编写配置。推荐安装 Neovim（sudo pacman -S neovim）使用 ~/.config/nvim/init.vim（向后兼容 Vimscript）或 init.lua（Lua 语法）。
~/.vimrc 配置语法详解： 
" ===== 基础设置 =====
set number              " 显示行号
set relativenumber      " 相对行号（方便 j/k 跳转时知道距离）
set tabstop=4           " Tab 显示为 4 个空格宽度
set shiftwidth=4        " 自动缩进 4 个空格
set expandtab           " 将 Tab 转换为空格
set autoindent          " 自动缩进
set smartindent         " 智能缩进（根据语法自动调整）
set cursorline          " 高亮当前行
set showmatch           " 高亮匹配的括号
set hlsearch            " 高亮搜索结果
set incsearch           " 增量搜索（边输入边搜索）
set ignorecase          " 搜索忽略大小写
set smartcase           " 有大写字母时区分大小写
set mouse=a             " 启用鼠标支持（点击定位、滚轮滚动）
set clipboard=unnamedplus " 使用系统剪贴板（可在 Vim 和其他应用间复制）
set encoding=utf-8      " 文件编码
set termguicolors       " 启用真彩色
set splitbelow          " 水平分屏时新窗口在下
set splitright          " 垂直分屏时新窗口在右
set scrolloff=8         " 光标距离屏幕边缘至少 8 行
set updatetime=300      " 交换文件写入间隔(ms)

" ===== 按键映射 =====
" 将 leader 键设为空格（方便快捷组合键）
let mapleader = " "

" 分屏导航（Ctrl+hjkl 在分屏间移动）
nnoremap <C-h> <C-w>h
nnoremap <C-j> <C-w>j
nnoremap <C-k> <C-w>k
nnoremap <C-l> <C-w>l

" 使用 jk 代替 Esc（更快退出插入模式）
inoremap jk <Esc>

" Leader+w 保存文件
nnoremap <leader>w :w<CR>

" Leader+q 退出
nnoremap <leader>q :q<CR>

" ===== 外观 =====
" 配色方案
colorscheme desert       " 内置主题（还可用：elflord、ron、slate、murphy 等）

" 状态栏
set laststatus=2         " 始终显示状态栏
set showmode             " 显示当前模式
set showcmd              " 显示正在输入的命令

" ===== 文件管理 =====
set undofile             " 持久化撤销历史（即使关闭文件也能撤销）
set undodir=~/.vim/undo  " 撤销文件存放目录
set backup               " 启用备份
set backupdir=~/.vim/backup
set noswapfile           " 不使用交换文件（现代编辑器习惯）
为什么这样配置（每个选项的意义）：
  • relativenumber — 显示相对行号，配合 10j 可直接跳过 10 行，无需心算目标行号
  • expandtab + tabstop=4 — 统一使用空格缩进，避免跨编辑器时 Tab/空格混用导致的对齐混乱
  • mouse=a — 在 Vim 中启用鼠标，新手友好，可点击定位光标、拖拽选择、滚轮翻页
  • clipboard=unnamedplus — Vim 的 yank/put 与系统剪贴板互通，可在 Vim 和浏览器间直接复制粘贴
  • hlsearch + incsearch — 搜索时实时高亮所有匹配项，快速定位目标
  • jk 映射 Esc — 退出插入模式无需将手移到 Esc 键，保持手指在键盘主区，效率翻倍
  • leader 键设为空格 — 空格是最易按的键，作为快捷键前缀后大量操作触手可及
  • undofile — 关闭文件后仍可撤销之前的编辑（通过 u），防止意外保存错误内容
推荐使用 Neovim 替代传统 Vim：sudo pacman -S neovim。配置文件为 ~/.config/nvim/init.lua（Lua 语法更现代易维护），而且大部分 Vimscript 语法仍兼容。若想快速上手，可参考社区配置如 NvChad 或 LazyVim。
系统 网络配置
Arch Linux 使用 NetworkManager 或 systemd-networkd 管理网络。NetworkManager 是桌面用户的推荐选择。
NetworkManager 常用命令：
  • nmtui — 终端图形化网络管理界面
  • nmcli device status — 查看所有网络设备状态
  • nmcli device wifi list — 扫描可用 WiFi
  • nmcli device wifi connect "SSID" password "密码" — 连接 WiFi
  • nmcli connection show — 查看已保存的连接
  • nmcli connection up/down "连接名" — 启用/禁用连接
  • ping archlinux.org — 测试网络连通性
  • ip addr / ip link — 查看 IP 地址和网络接口
系统 切换软件源与更新密钥
切换镜像源（提速下载）：
编辑 /etc/pacman.d/mirrorlist，将速度最快的镜像放在文件顶部。
推荐使用 reflector 自动选择最快镜像： 
# 安装 reflector
sudo pacman -S reflector

# 自动选择中国地区最快的 10 个镜像并更新配置
sudo reflector --country China --age 12 --protocol https --sort rate --save /etc/pacman.d/mirrorlist --latest 10

# 也可手动编辑 mirrorlist
sudo nano /etc/pacman.d/mirrorlist
更新密钥环（解决 GPG 签名错误）：
当出现 "signature is unknown trust" 或密钥过期错误时： 
# 更新 archlinux-keyring
sudo pacman -S archlinux-keyring

# 初始化 pacman 密钥
sudo pacman-key --init
sudo pacman-key --populate archlinux

# 刷新所有密钥
sudo pacman-key --refresh-keys
系统 系统更新策略
Arch Linux 采用滚动更新模型。推荐每周更新一次，同时做好快照备份。
标准更新流程： 
# 全面更新（官方库 + AUR）
sudo pacman -Syu            # 仅官方库
yay -Syu                    # 含 AUR 的全面更新

# 清理旧包缓存（保留最近 3 个版本）
sudo paccache -r
# 需先安装：sudo pacman -S pacman-contrib

# 清理孤立依赖
sudo pacman -Rns $(pacman -Qdtq)
更新前务必查看 Arch Linux 官方新闻，确认是否有需要手动干预的变更。内核或显卡驱动大版本更新后建议重启。
系统 btrfs 快照与系统回滚
btrfs 是现代化的写时复制（CoW）文件系统，支持快照、子卷、压缩等高级特性。配合 snapper 或 timeshift 可实现系统快照和回滚。
手动快照操作： 
# 创建只读快照
sudo btrfs subvolume snapshot -r /mnt/@ /mnt/@snap-$(date +%Y%m%d)

# 列出所有子卷
sudo btrfs subvolume list /

# 回滚：将快照设为默认子卷
sudo btrfs subvolume set-default <snapshot-id> /
# 然后重启即可进入快照状态
自动快照 — 使用 snapper： 
# 安装
sudo pacman -S snapper snap-pac

# 为根子卷创建配置
sudo snapper -c root create-config /

# 设置自动快照策略（每小时 + 每天 + 每周）
sudo snapper -c root set-config "TIMELINE_CREATE=yes"
sudo snapper -c root set-config "TIMELINE_CLEANUP=yes"

# 手动创建快照
sudo snapper -c root create -d "更新前备份"

# 列出快照
sudo snapper -c root list

# 回滚到指定快照
sudo snapper -c root undochange 5..0
使用 Timeshift（图形化快照工具）：
yay -S timeshift — 适合不熟悉命令行的用户，提供 GUI 界面管理 btrfs 快照。

平铺 niri — 可滚动平铺窗口管理器
niri 是一款创新的 Wayland 平铺窗口管理器，采用"无限水平滚动"的工作区设计。所有窗口排列在一行，类似于横向滚动的页面。
安装： 
sudo pacman -S niri
# 或开发版本
yay -S niri-git
配置文件：
主配置位于 ~/.config/niri/config.kdl（使用 KDL 格式）。
默认配置生成：首次运行 niri 会自动生成默认配置。
关键配置项： 
// 输出设置
output "eDP-1" {
    mode 1920x1080@60Hz
    scale 1.0
}

// 快捷键绑定
binds {
    Mod+Q { close-window; }
    Mod+Return { spawn "kitty"; }
    Mod+D { spawn "fuzzel"; }          // 应用启动器
    Mod+Shift+E { quit; }
}

// 自动启动
spawn-at-startup "waybar"
spawn-at-startup "swaybg -i ~/wallpaper.jpg"
niri 官方文档：github.com/YaLTeR/niri/wiki
⌨ 在 niri 中安装和配置 fcitx5 输入法：
Wayland 环境下需要 fcitx5 作为输入法框架。安装后需配置环境变量和自启动。
安装 fcitx5： 
sudo pacman -S fcitx5 fcitx5-chinese-addons fcitx5-qt fcitx5-gtk fcitx5-configtool
在 niri 中配置输入法环境变量和自启动： 
// 在 config.kdl 中添加 fcitx5 环境变量
environment {
    GTK_IM_MODULE "fcitx"
    QT_IM_MODULE "fcitx"
    XMODIFIERS "@im=fcitx"
    SDL_IM_MODULE "fcitx"
    GLFW_IM_MODULE "ibus"
}

// 自启动 fcitx5
spawn-at-startup "fcitx5 -d"

// 输入法切换快捷键（在 fcitx5-configtool 中设置）
// 默认切换键为 Ctrl+Space，可在 fcitx5 配置工具中修改
配置输入法切换快捷键：
运行 fcitx5-configtool → 全局选项 →「切换激活/非激活输入法」设置你偏好的快捷键（如 Ctrl+Space 或 Shift+Space）。
在 fcitx5 的「附加组件」→「XKB 选项」中可以在中英文之间快速切换。
平铺 Hyprland — 动态平铺窗口管理器
Hyprland 是目前最热门的 Wayland 动态平铺窗口管理器，以流畅动画和高度可定制著称。
安装与启动： 
sudo pacman -S hyprland kitty waybar wofi
# 在登录管理器中选择 Hyprland，或从 tty 启动：
Hyprland
配置文件：
主配置：~/.config/hypr/hyprland.conf
键盘快捷键、窗口规则、动画、显示器配置等都在此文件中定义。
核心配置示例： 
# 显示器
monitor=eDP-1,1920x1080@60,0x0,1

# 快捷键
bind = SUPER, Q, killactive
bind = SUPER, RETURN, exec, kitty
bind = SUPER, D, exec, wofi --show drun
bind = SUPER, E, exec, thunar

# 自动启动
exec-once = waybar
exec-once = swaybg -i ~/wallpaper.jpg
exec-once = dunst

# 窗口规则
windowrulev2 = float, class:(pavucontrol)
windowrulev2 = tile, class:(kitty)
推荐配套应用启动器：
  • wofi — 轻量 Wayland 应用启动器（sudo pacman -S wofi）

平铺 fastfetch — 系统信息配置详解
fastfetch 是 neofetch 的现代替代品，用 C 语言编写，速度更快、功能更丰富。安装：sudo pacman -S fastfetch
配置文件位置：~/.config/fastfetch/config.jsonc（JSON with Comments 格式）
首次运行会自动生成默认配置：fastfetch --gen-config
config.jsonc 语法结构： 
{
    // === 全局显示设置 ===
    "$schema": "https://github.com/fastfetch-cli/fastfetch/raw/dev/doc/json_schema.json",
    "logo": {
        "type": "auto",           // Logo类型：auto / small / builtin / none / 自定义文件路径
        "padding": { "top": 2 }
    },
    "display": {
        "separator": " → ",       // 键与值之间的分隔符
        "color": { "keys": "38;5;110" },  // 键名颜色（ANSI 颜色码）
        "keyWidth": 14,           // 键名对齐宽度
        "size": { "binary": { "prefix": "IEC" } }  // 容量单位（IEC=KiB/MiB）
    },

    // === 要显示的模块及其顺序 ===
    "modules": [
        "title",                  // 标题行（用户名@主机名）
        "separator",              // 分隔线
        "os",                     // 操作系统
        "host",                   // 主机型号
        "kernel",                 // 内核版本
        "uptime",                 // 运行时间
        "packages",               // 已安装包数量
        "shell",                  // Shell 版本
        "display",                // 显示器信息
        "de",                     // 桌面环境
        "wm",                     // 窗口管理器
        "wmtheme",                // WM 主题
        "theme",                  // 系统主题
        "icons",                  // 图标主题
        "font",                   // 字体
        "cursor",                 // 光标
        "terminal",               // 终端
        "terminalfont",           // 终端字体
        "cpu",                    // CPU
        "gpu",                    // GPU
        "memory",                 // 内存
        "swap",                   // 交换分区
        "disk",                   // 磁盘
        "battery",                // 电池
        "locale",                 // 语言区域
        "localip",                // 本地 IP
        "media",                  // 正在播放的媒体
        "break",                  // 空行
        "colors"                  // 终端颜色块展示
    ]
}
常用配置技巧与原因：
  • 选择 logo 类型："type": "small" 用简版 logo 节省空间；"type": "none" 纯文本输出，适合截图分享
  • modules 数组控制显示顺序：无关信息可移除（如不玩媒体则不显示 media），加快启动速度
  • separator 分隔符：用 → 而非 : 增强视觉层次感
  • keyWidth：统一键名对齐，让信息排列整齐易于扫描
  • binary prefix：IEC (1024进制) 更精确；SI (1000进制) 更符合厂商标注习惯
  • ANSI 颜色码：38;5;N 为 256 色调色板，可选任何颜色；如 38;5;110 是柔和的蓝色
使用 fastfetch -c all 可以临时查看所有可用模块的完整信息，方便决定哪些模块对你有用。
平铺 quickshell — Shell 配置框架详解
quickshell 是一个为 zsh 设计的现代配置框架，类似于 oh-my-zsh 但更轻量快速。它管理插件、主题、别名和环境变量。
安装： 
# 确保已安装 zsh
sudo pacman -S zsh

# 安装 quickshell
yay -S quickshell-git

# 切换到 zsh（如果还没切换）
chsh -s /bin/zsh
配置文件：~/.config/quickshell/quickshell.toml（TOML 格式）
首次安装后运行 quickshell init 自动生成默认配置。
quickshell.toml 语法结构： 
# ===== 通用设置 =====
[general]
theme = "catppuccin"              # 内置主题名
prompt_style = "powerline"        # 提示符风格：powerline / plain / minimal
enable_syntax_highlighting = true  # 命令语法高亮
enable_autosuggestions = true     # 自动补全建议

# ===== 环境变量 =====
[env]
EDITOR = "nvim"
VISUAL = "nvim"
BROWSER = "firefox"

# ===== 别名 =====
[aliases]
ll = "ls -lah"
la = "ls -A"
update = "yay -Syu"
clean = "sudo pacman -Rns $(pacman -Qdtq)"
ip = "ip -color addr"

# ===== 插件 =====
[plugins]
enabled = [
    "git",           # Git 别名和分支显示
    "docker",        # Docker 补全和别名
    "sudo",          # 按两次 ESC 在命令前插入 sudo
    "colored-man",   # man 手册页彩色输出
    "extract",       # 万能解压命令（支持所有压缩格式）
    "fzf",           # 模糊搜索集成
    "z",             # 智能目录跳转
    "ssh-agent"      # SSH 代理自动管理
]

# 插件特定配置
[plugins.git]
show_untracked = true
show_stash = true

# ===== 主题配置 =====
[theme]
# 自定义 Catppuccin 主题颜色（覆盖默认值）
colors = {
    primary = "#89b4fa",
    secondary = "#a6e3a1",
    error = "#f38ba8",
    warning = "#fab387",
    info = "#89dceb"
}

# ===== 自定义函数 =====
[functions]
# 项目快速跳转
project = "cd ~/Projects/$(ls ~/Projects | fzf)"
# Git 一键推送
gp = "git add . && git commit -m \"$1\" && git push"
为什么要这样配置：
  • prompt_style = "powerline" — 提供当前目录、Git 分支、命令状态等信息，减少频繁使用 pwd 和 git status
  • syntax_highlighting — 输入命令时实时高亮有效命令（绿色）/ 无效命令（红色），减少拼写错误
  • autosuggestions — 基于历史记录自动提示灰色建议文本，按 → 键补全，大幅提高输入速度
  • fzf 插件 — Ctrl+T 模糊搜索文件、Ctrl+R 搜索历史命令，无需记忆完整路径
  • extract 插件 — 一个 extract 文件名 命令自动识别格式并解压，无需记忆 tar 参数
  • z 插件 — 根据使用频率自动学习，z project 即可跳转到常去目录
  • aliases — 高频长命令缩写成短别名，每天节省数百次按键

屏幕模式说明：
  • 扩展（Extend） — 画面延伸到第二屏幕，两个屏幕显示不同内容，鼠标可跨屏移动。这是最常用的多屏模式。
  • 复制（Clone/Mirror） — 两个屏幕显示完全相同的画面。适合投影/演示场景。
  • 仅主屏幕 — 禁用外接显示器，只使用内置屏幕。
  • 仅外接屏幕 — 关闭内置屏幕，只使用外接显示器。笔记本合盖外接显示器时常用。
快捷切换：
快捷键 Meta+P（部分系统）或使用 KDE 面板中的显示切换小部件。
也可在系统托盘的「显示」图标中快速切换模式。
调整显示器布局时，可以设置主显示器（勾选"主显示器"复选框），任务栏和通知将优先显示在主屏上。
